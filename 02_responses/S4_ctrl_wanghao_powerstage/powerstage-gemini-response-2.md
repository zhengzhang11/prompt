关于你的四个疑问，解答如下（直接点明操作）：

---

# 1. 连到转矩源的“黑线”是从哪里来的？

- **来源**：  
  这根黑线代表负载阻力转矩（$T_{load}$）

- **你的操作**：  
  1. 在界面中放一个 **Inport（输入端口）模块**（位于 Simulink / Ports & Subsystems）  
  2. 重命名为 `T_load`  
  3. 将其黑线连接到 **Simulink-PS Converter**

- **后续**：  
  - 当你把模型交给 A 同学时：  
    - A 会将 S2 小组的阻力模型输出直接连接到 `T_load`  
  - 自测方法：  
    - 可使用 **Step** 或 **Constant** 模块提供测试数值  

---

# 2. PMSM 的 n 接口怎么接？

- **操作**：  
  - 直接悬空（不连接任何东西）

- **原因**：  
  - `n` 表示电机三相星型绕组的中性点（Neutral）  
  - 当前系统为三相无刷驱动，仅使用 A / B / C 三相输出  
  - 无需零线，因此该接口不连接  

---

# 3. 你的部分输出是哪根线？（需要交付的内容）

由于使用的是 **Foundation Library（底层物理库）**，电机不会自动输出信号，需要手动添加传感器。

---

## 输出一：转速与位置（提供给 A 做速度环）

### 操作步骤：

1. 添加 **Ideal Rotational Motion Sensor（理想旋转运动传感器）**
2. 连接方式：
   - `R` → 电机 `R`（并联）
   - `C` → Mechanical Rotational Reference（机械地）

3. 输出信号：
   - `W`（角速度）
   - `A`（角度）

4. 信号转换：
   - 分别接入 **PS-Simulink Converter**

5. 输出：
   - 使用 **Outport** 导出两根黑线：
     - `Omega`（角速度）
     - `Theta`（角位置）

---

## 输出二：相电流（提供给 A 做电流环）

### 操作步骤：

1. 添加 **Current Sensor（电流传感器）**

2. 接入方式（串联）：
   - 将逆变器 → 电机 A 相的连接线断开  
   - 插入传感器：
     - 逆变器侧 → `+`
     - 电机侧 → `-`

3. 信号输出：
   - `I` 端口 → **PS-Simulink Converter**

4. 输出：
   - 使用 **Outport** 导出：
     - `I_a`

5. 扩展：
   - 若需要两相电流（如 A 同学要求）  
   - 在 B 相重复上述步骤即可  

---

# 4. Solver Configuration 连在电压源正极可以吗？

- **结论**：  
  ✔ 完全可以，而且是推荐做法  

- **原因**：  
  - Solver Configuration 的作用是：  
    > 声明该物理网络需要特殊数值求解  

  - 它可以连接在任意蓝色电学网络节点上  
  - 接在电源正极或负极都符合规范  

- **建议设置**：  
  - 双击模块  
  - 勾选 `Use local solver`  
  - 或根据系统需要调整步长参数  

---
