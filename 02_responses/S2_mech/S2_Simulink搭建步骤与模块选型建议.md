# S2 Simulink 搭建步骤与模块选型建议

> 修订依据：S1 顶层文档《构架方案3（被控对象修正）》v1.5 + 题目原始 PDF  
> 当前结论：S2 在 Simulink 中不仅负责 mechanical_load，还负责 L2 电机与传动层的对象建模与参数化  
> S2 建模边界：BLDC 电机对象参数化、行星减速机、机械负载支路、最小测试台  
> 不在本文范围内：控制器 PI、PWM、逆变器桥、硬件保护逻辑

---

## 一、S2 在 Simulink 中到底负责什么

根据 S1 v1.5 顶层文档第 0 章和第 2 章，S2 对应 L1 工况与负载层 + L2 电机与传动层，因此在 Simulink 中应交付的不只是 mechanical_load，而是下面三部分：

1. BLDC 电机本体参数化后的对象层。
2. 12.5:1 行星减速机与机械传动层。
3. mechanical_load 负载计算与加载支路。

因此，S2 在模型里的责任不是控制，也不是功率电子，而是把“被控对象的机械和电机主体”搭清楚，供 S4 后续闭环接入。

---

## 二、S2 应交付的模块清单

| 模块 | 是否必须 | 作用 | 推荐实现 |
|------|---------|------|---------|
| BLDC 电机本体 | 必须 | 建立 57BLF02 电机对象 | 优先调用官方示例或库中 BLDC 块，参数化 |
| 行星减速机 | 必须 | 建立 i=12.5、η=0.90 的传动层 | Simple Gear |
| mechanical_load | 必须 | 根据坡度、质量、速度生成负载转矩 | Subsystem + MATLAB Function |
| 负载施加支路 | 必须 | 把负载力矩施加到机械网络 | Controlled Ideal Torque Source |
| 速度测量支路 | 强烈建议 | 输出 omega_m、omega_w 供验证和交接 | Ideal Rotational Motion Sensor + PS-Simulink Converter |
| S2 最小测试台 | 强烈建议 | 独立验证电机传动对象和负载支路 | 独立 testbench |

## 三、模块选型结论

### 3.1 电机本体怎么搭

结论：

优先调用 MathWorks 官方 BLDC Speed Control 示例里的电机对象，或直接调用库中的 BLDC 电机块，不建议手工写状态方程。

原因：

1. S1 已经明确本项目核心模板是 BLDCSpeedControlExample。
2. 57BLF02 只是参数替换，不需要重写电机方程。
3. 手工写电机对象容易在反电势、换相和 Hall 相位上出错。
4. 课程重点是对象建模和系统集成，不是从零复现电机理论块图。

### 3.2 减速机怎么搭

结论：

用 Simple Gear，不手工写齿轮效率模型。

原因：

1. S1 已明确指定 Simscape 的 Simple Gear。
2. 只需要录入减速比和效率即可。
3. 这是最稳妥的工程实现方式。

### 3.3 mechanical_load 怎么搭

结论：

用 Subsystem + MATLAB Function Block 手工搭。

原因：

1. 这是 S2 自定义逻辑，不是现成标准对象。
2. 公式集中，最便于以后扩展速度阻尼项。
3. 交给 S4 时接口最清楚。

### 3.4 负载力矩怎么接到机械网络

结论：

用 Controlled Ideal Torque Source，把 mechanical_load 计算出的力矩以物理方式加到机械轴上。

原因：

1. 这样 gear、motor、load 全部在同一个物理旋转网络内。
2. 不需要手写机械方程块来做力矩减项。
3. 更符合 S1 所说的“电机输出轴 + Simple Gear + 机械负载子系统”的实现思路。

## 四、最重要的规则：不要双重折算

如果你在 Simscape 里已经放了 Simple Gear，就不能再把已经折算到电机轴侧的 T_L_m 当作唯一物理加载量再施加到同一物理链上，否则会把减速机折算重复算一遍。

因此本文明确规定：

1. mechanical_load 仍然计算并输出 T_L_m 和 T_L_w 两个量。
2. 在“带 Simple Gear 的物理模型”中，真正接到 Controlled Ideal Torque Source 上的应是 T_L_w，也就是轮侧负载力矩。
3. T_L_m 保留为观测量、校核量和给 S4 的接口说明量。
4. 如果未来 S4 采用纯数学简化机械方程、不保留物理齿轮，则才使用 T_L_m 直接进入机械方程。

这条规则必须写死，否则模型一定会出现重复折算。

## 五、S2 推荐交付结构

建议 S2 最终交付三个模型层次：

1. mechanical_load.slx
2. s2_motor_mech_unit.slx
3. s2_motor_mech_testbench.slx

其关系如下：

1. mechanical_load.slx 只负责把 slope_deg、mass_kg、omega_m 转成 T_L_w 和 T_L_m。
2. s2_motor_mech_unit.slx 负责把 BLDC 电机、Simple Gear、负载施加支路组合起来。
3. s2_motor_mech_testbench.slx 负责独立验证 S2 模块是否接线正确、参数是否正确。

## 六、第一部分：mechanical_load.slx 的具体搭建步骤

### 6.1 第 1 步：新建模型

1. 打开 Simulink。
2. 新建 Blank Model。
3. 立即保存为 mechanical_load.slx。
4. 在仿真前确保已经运行 init_params.m。

### 6.2 第 2 步：放输入端口

从库里拖出 3 个 Inport，依次命名为：

1. omega_m
2. slope_deg
3. mass_kg

### 6.3 第 3 步：放输出端口

从库里拖出 2 个 Outport，依次命名为：

1. T_L_m
2. T_L_w

### 6.4 第 4 步：放核心计算块

从库里拖出 1 个 MATLAB Function，命名为：

load_torque_calc

### 6.5 第 5 步：接线

按以下顺序接线：

1. omega_m 接到 load_torque_calc 第 1 输入。
2. slope_deg 接到 load_torque_calc 第 2 输入。
3. mass_kg 接到 load_torque_calc 第 3 输入。
4. load_torque_calc 第 1 输出接 T_L_m。
5. load_torque_calc 第 2 输出接 T_L_w。

### 6.6 第 6 步：先声明 MATLAB Function 参数

这一步不能省。

仅仅运行 `init_params.m` 还不够，因为 MATLAB Function 块在编译时不会自动把 base workspace 里的任意变量当作可用参数。

你必须先在 `load_torque_calc` 这个 MATLAB Function 块里，把下面 5 个量显式声明为参数：

1. `g`
2. `f_roll`
3. `r`
4. `i_ratio`
5. `eta_m`

具体操作：

1. 双击打开 `load_torque_calc`。
2. 在 MATLAB Function 编辑器里打开 `Symbols Pane`，如果没显示，就在编辑器菜单里打开符号窗格。
3. 依次新增 `g`、`f_roll`、`r`、`i_ratio`、`eta_m`。
4. 把它们的 `Scope` 全部设为 `Parameter`。
5. 保持类型为标量 `double`。

做完这一步后，MATLAB Function 块才会去绑定你在 `init_params.m` 里初始化出来的同名变量。

### 6.7 第 7 步：写入代码

将 MATLAB Function 代码替换为：

```matlab
function [T_L_m, T_L_w] = load_torque_calc(omega_m, slope_deg, mass_kg)
%#codegen

N_1_alpha = (mass_kg * g / 2) * cosd(slope_deg);
F_g_1     = (mass_kg * g / 2) * sind(slope_deg);
F_r_1     = f_roll * N_1_alpha;

T_L_w = (F_r_1 + F_g_1) * r;

% 当前版本预留速度接口，不引入速度相关阻尼
T_vis = 0 * omega_m;

T_L_m = T_L_w / (i_ratio * eta_m) + T_vis;
end
```

### 6.8 第 8 步：为什么你会报“未定义函数或变量 g”

根因是：

1. `init_params.m` 确实把 `g` 放进了 base workspace。
2. 但 MATLAB Function 块不是普通 MATLAB 脚本，它要先经过 Simulink Coder 风格的编译分析。
3. 对这个块来说，只有下面几类量是合法可见的：输入端口、块内部局部变量、显式声明的 Parameter、或显式声明的数据对象。
4. 你现在代码里直接写 `g`，但块里没有把 `g` 声明为 Parameter，所以编译器认为它未定义。

因此：

“我已经运行了初始化脚本”这件事本身没有错，但对 MATLAB Function 块来说还差一步，就是把这些工作区变量声明成块参数。

### 6.9 第 9 步：本模块验收值

1. slope_deg = 0，mass_kg = 50，omega_m = 0 时，应有 T_L_w ≈ 0.785，T_L_m ≈ 0.0698。
2. slope_deg = 5，mass_kg = 50，omega_m = 0 时，应有 T_L_w ≈ 2.492，T_L_m ≈ 0.2215。

---

### 6.10 第 10 步：如果你不想声明 Parameter，还有两个替代方案

替代方案 A：把 `g`、`f_roll`、`r`、`i_ratio`、`eta_m` 也做成输入端口。

优点：最直接。  
缺点：端口太多，不利于后续交付。

替代方案 B：在函数里临时写死常数。

例如：

```matlab
g_local = 9.81;
f_local = 0.04;
r_local = 0.08;
i_local = 12.5;
eta_local = 0.90;
```

优点：马上能跑。  
缺点：违背参数单一入口原则，不推荐作为正式版本。

正式版本仍然建议使用“`init_params.m` + MATLAB Function 参数声明”的方式。

## 七、第二部分：s2_motor_mech_unit.slx 的具体搭建步骤

这一部分是本次修订的核心。S2 现在不仅交付负载计算，还要交付电机与传动对象层。

### 7.1 第 1 步：新建对象层模型

1. 新建 Blank Model。
2. 保存为 s2_motor_mech_unit.slx。
3. 运行 init_params.m。
4. 如果你准备从官方示例复制电机对象，先执行：

```matlab
openExample('simscapeelectrical/BLDCSpeedControlExample')
```

### 7.2 第 2 步：放 BLDC 电机本体

推荐做法有两种，优先级如下：

1. 优先从官方示例中复制电机对象部分到你的模型。
2. 若不方便复制，则在 Library Browser 中搜索 BLDC Motor 或 Brushless DC Motor，使用等效电机块。

要求：

1. 电机块必须是可参数化的对象块，不要自己用 Integrator 重写。
2. 电机参数必须录入为 57BLF02 冻结值。

应录入的关键参数至少包括：

1. 额定电压 24V。
2. 额定功率 125W。
3. 额定转速 3000rpm。
4. Kt = 0.066。
5. Ke = 6.3 V/krpm 对应的 SI 换算值。
6. Rs = 0.30Ω。
7. Ls = 0.75mH。
8. Jm = 1.7e-5 kg·m²。
9. 极对数 p = 4。

### 7.2.1 你截图里的 BLDC Motor 参数框应如何填写

下面按你截图里这个参数窗口的栏目顺序说明。

#### A. 顶部通用项

| 参数项 | 建议填写 | 说明 |
|------|---------|------|
| 构建选项 | 无热端口 | 第一版模型不做热网络，先保证对象层跑通 |
| 所选部件 | 留空 | 除非你在用 Simscape 组件库关联部件，否则不用点这个 |

#### B. 转子页签

| 参数项 | 建议填写 | 说明 |
|------|---------|------|
| 电气连接 | 复合三相端口 | 若你后面准备接官方示例或三相桥，这个最方便 |
| 反电动势曲线 | 理想梯形 - 指定最大磁链 | 符合 57BLF02 的 BLDC 梯形反电势建模口径 |
| 最大永磁磁链 | 0.03 Wb | 这是由 `Ke` 映射来的块内参数，可先按 0.03 作为工程初值 |
| 反电动势恒定的转子角 | pi/12 rad | 先保持这个值，与六步换相模板兼容；若后续 Hall 相位错位再微调 |
| 极对数 | 4 | 冻结值，8 极电机对应 4 对极 |
| 转子角定义 | a 相磁轴和 d 轴之间的角度 | 保持截图当前选项即可 |

关于“最大永磁磁链”要特别说明：

1. 这个量不是 `params.md` 里直接给的铭牌量。
2. 它是该 BLDC 块内部为了生成梯形反电势所使用的等效参数。
3. 当前工程上可先用 0.03 Wb 作为 57BLF02 的初始值。
4. 后续若发现 3000 rpm 空载时反电势明显不对，再围绕这个量微调，而不是先改 `Ke` 冻结值。

#### C. 定子页签

这里最容易填错。

`init_params.m` 里的 `Rs=0.30 Ω`、`Ls=0.75 mH` 是按线线等效、两相串联导通口径冻结的；而很多电机块的定子参数栏要求的是单相参数。

因此你要按下面规则处理：

| 冻结量 | 冻结值 | 若块要求单相参数时应填写 |
|------|------|----------------------|
| 线线电阻 `Rs` | 0.30 Ω | 相电阻 `R_phase = 0.15 Ω` |
| 线线电感 `Ls` | 0.75 mH | 相电感 `L_phase = 0.375 mH` |

推荐填写口径：

1. 如果块参数名称写的是“相电阻”“每相电阻”“Phase resistance”，填 `0.15`。
2. 如果块参数名称写的是“相电感”“每相电感”“Phase inductance”，填 `0.375e-3`。
3. 如果块里明确写的是线线参数或两相导通等效参数，才填 `0.30` 和 `0.75e-3`。

第一版最稳妥的经验规则是：

1. 电机对象块通常更偏向单相参数。
2. 控制器与推导文档里的 `Rs/Ls` 更偏向线线等效参数。
3. 两者不要直接混填。

如果你的界面和截图一致，建议直接这样填：

| 字段名 | 建议填写 | 原因 |
|------|---------|------|
| 绕组类型 | 星型绕组 | 57BLF02 按三相星形口径建模最合理 |
| 定子参数化 | 指定 Ld、Lq 和 L0 | 与该块当前建模方式一致 |
| 建模保真度 | 常数 Ld 和 Lq | 第一版先用常值模型，避免复杂化 |
| 定子 d 轴电感 Ld | 0.000375 H | 由冻结线线电感 0.75 mH 按单相等效折半得到 |
| 定子 q 轴电感 Lq | 0.000375 H | 第一版按表贴式/近似非凸极电机取 Ld=Lq |
| 定子零序电感 L0 | 0.000375 H | 若保留零序项但无额外数据，先与 Ld、Lq 取同值 |
| 每相定子电阻 Rs | 0.15 Ohm | 由冻结线线电阻 0.30 Ω 折算到单相 |
| 零序 | 建议改成“不包括”或“排除” | 三线制星形绕组第一版通常不需要显式零序通道 |

你截图里当前这些值不建议继续用：

1. `Ld = 0.00022 H`
2. `Lq = 0.00022 H`
3. `L0 = 0.00016 H`
4. `Rs = 0.013 Ohm`

原因是这些数值明显更像示例默认值，不是 57BLF02 当前冻结参数口径。

关于“零序”这一项，建议优先这样处理：

1. 如果下拉框允许改成“不包括/排除”，就直接改掉，这是第一推荐。
2. 如果你暂时保留“包括”，那就把 `L0` 先填成 `0.000375 H`，不要继续保留 `0.00016 H`。

这样做的原因是：

1. 当前题目和冻结参数没有给出独立零序参数。
2. 第一版对象层目标是先得到一个与冻结值一致、可稳定运行的 BLDC 对象。
3. 零序建模不是当前课程设计的重点。

如果你的定子页签里还有“互感”项：

1. 第一版可先填 0。
2. 先把对象跑通，再考虑是否需要引入相间互感。

#### D. 铁损页签

如果有“启用铁损”或类似选项：

1. 第一版建议关闭，或者使用默认的无铁损近似。
2. 原因是当前冻结参数并没有独立给出铁损系数。
3. 若你在对象层就把铁损加进去，容易与后续效率分析重复口径。

#### E. 机械页签

| 参数项 | 建议填写 | 说明 |
|------|---------|------|
| 转子惯量 | 1.7e-5 kg·m² | 直接填冻结值 `J_m` |
| 粘性摩擦/机械阻尼 | 0 或极小值 | 第一版建议填 0，避免和外部负载、减速机损耗重复 |

这里的关键原则是：

1. 机械负载已经由 external load 支路处理。
2. 减速机损耗已经由 gear efficiency 处理。
3. 因此电机块内部若再加明显机械阻尼，会造成重复损耗。

#### F. 初始目标页签

第一版建议：

1. 初始角速度设 0。
2. 初始角位置设 0。
3. 初始电流设 0。

保持最干净的启动条件，方便后续定位问题。

#### G. 标称值页签

这一页通常不是“必须精确建模参数”，更多是数值求解和单位缩放参考。

要特别说明：

1. 这个 BLDC 块的“标称值”页不一定会出现“标称电压”字段。
2. 你应以当前界面实际出现的字段为准，不要为了找“标称电压”去怀疑块选错了。
3. 对这个块来说，没有“标称电压”输入框是正常现象。

如果这页不填也能跑，不必在第一版死抠。

如果你的界面和截图一致，标称值页建议这样处理：

| 字段 | 是否勾选 | 建议值 | 说明 |
|------|---------|------|------|
| D 轴电流 | 不勾选，或勾选后填 0 A | 0 | 第一版 BLDC 六步换相不需要重点使用 d 轴电流标称值 |
| Q 轴电流 | 可不勾选；若勾选填 7.8 A | 7.8 | 用额定电流作为典型量级最合理 |
| 转矩 | 建议勾选 | 0.4 N·m | 用额定转矩作为标称值 |
| 转子转速 | 建议勾选 | 314.16 rad/s | 对应 3000 rpm |
| 转子角 | 一般不勾选；若必须填可设 1 rad | 1 | 这项通常不是第一版关键项 |

对应到你截图里的最简建议是：

1. 保持“转矩”勾选。
2. 保持“转子转速”勾选。
3. “D 轴电流”先不要勾。
4. “Q 轴电流”可先不勾；若你希望数值缩放更稳，可以勾上并填 `7.8`。
5. “转子角”先不要勾。

所以第一版最稳妥的标称值配置是：

1. 转矩 = `0.4 N·m`
2. 转子转速 = `314.16 rad/s`
3. 其余项先不启用

如果后续仿真收敛性不好，再补充：

1. Q 轴电流 = `7.8 A`
2. D 轴电流 = `0 A`

### 7.2.2 一份可直接照填的 57BLF02 参数表

如果你的块参数页和截图一致，可按下面这份表填写：

| 页面 | 参数 | 建议值 |
|------|------|------|

2. `Friction model = Constant efficiency`，这项正确。
3. `Efficiency = 0.9`，这项正确。
4. `Backlash` 第一版不启用，正确。
5. `Viscous friction coefficients = [0, 0]`，第一版正确，避免与外部损耗重复。

### 7.3.2 是否会和 mechanical_load 里的效率重复计算

结论分两种情况：

#### 情况 A：你把 `T_L_w` 接到轮侧的 Controlled Ideal Torque Source

这时 **不会** 因为 gear 自身的 `Efficiency = 0.9` 而重复计算效率。

原因：

1. `T_L_w` 是轮侧物理负载力矩。
2. 轮侧负载通过物理齿轮传回电机轴时，齿轮效率 0.9 只在 gear 模块里生效一次。
3. 此时 `mechanical_load` 里虽然也算了一个 `T_L_m = T_L_w/(i_ratio*eta_m)`，但只要你没有把这个 `T_L_m` 再施加到机械网络，它就只是观测量，不会导致重复损耗。

#### 情况 B：你把 `T_L_m` 直接接到了机械轴负载上，同时模型里又保留了 Simple Gear

这时 **会重复计算**。

原因：

1. `T_L_m` 已经在 `mechanical_load` 里除过一次 `i_ratio * eta_m`。
2. 你又保留了物理齿轮的效率 0.9。
3. 这样等于“数学折算一次 + 物理齿轮再损失一次”，会把效率算两遍。

因此最终规则只有一句话：

**保留 Simple Gear 时，物理加载用 `T_L_w`；`T_L_m` 只看不接。**

### 4.1 你现在选择的简化路线是什么

如果你决定：

1. 继续使用 `mechanical_load` 里已经折算好的 `T_L_m`
2. 不再保留 Simscape 里的 `Simple Gear`

那么这是完全可行的简化方案。

这条路线的本质是：

1. 减速比 `i_ratio = 12.5` 和机械效率 `eta_m = 0.90` 不再由物理 gear 模块体现。
2. 它们已经提前被写进了 `mechanical_load` 的折算公式里。
3. 机械网络中直接对电机轴施加等效负载 `T_L_m`。

### 4.2 两条路线必须二选一

后续建模只能保留下面两种路线之一，不能混用：

| 路线 | 是否保留 Simple Gear | 施加的负载量 |
|------|----------------------|-------------|
| 物理传动路线 | 保留 | `T_L_w` |
| 简化等效路线 | 删除 | `T_L_m` |

你现在既然决定走简化路线，后续就应统一采用：

1. 删除 `Simple Gear`
2. 删除 gear 的 `Efficiency = 0.9` 这层物理损耗
3. 直接把 `T_L_m` 接到电机轴侧负载支路

### 4.3 这条简化路线的优缺点

优点：

1. 接线更简单。
2. 不会再遇到 gear 方向、效率重复计算的问题。
3. 更适合先把对象层快速跑通。

缺点：

1. 模型里不再显式存在物理减速机模块。
2. 你看不到 `omega_w = omega_m / 12.5` 这样的物理传动关系。
3. 后续若要展示传动层结构，报告里要说明“此处采用轴侧等效模型”。

### 7.3.3 你这张截图里还有一个需要注意的地方：旋转方向

你当前截图显示：

`Output shaft rotates = In opposite direction to input shaft`

这对“普通外啮合两齿轮”是合理的，但对“行星减速机”未必是最符合物理直觉的设置。

当前项目里你选的是行星减速机，通常希望输入轴和输出轴方向保持一致。因此建议你检查这个下拉框：

1. 如果有“与输入同方向”选项，优先改成同方向。
2. 如果这个块只能用反方向，那也不是绝对不能用，但你后续必须统一符号定义，否则轮速方向、负载转矩方向和控制参考方向容易全反。

## 十二、求解器刚性问题与三级修复方案

> 新增时间：2026-04-19  
> 问题来源：powerstage 仿真在 t ≈ 0.901094 s 处触发求解器警告

### 12.1 问题现象

Simulink 在仿真运行时报告：

> 在时间 0.901094 处，求解器无法在不违反最小步长 3.20133E-15 的情况下连续 1 次减小步长。求解器将继续仿真，步长限制为 3.20133E-15 并使用有效相对误差容限 0.0014832，该容限大于指定的相对误差容限 0.001。

步长 3.20133E-15 已接近双精度浮点机器精度（约 2.22E-16），求解器事实上已经失效。

### 12.2 根本原因

本项目存在两个刚性来源叠加的情况：

1. **高频 PWM 开关**：PWM Generator 工作在 20 kHz（T_s = 50 μs），每次开关跳变会对 Simscape 电气网络注入瞬时不连续激励。
2. **Simscape 物理网络的 DAE 约束**：BLDC 电机模型内部含有代数约束方程（DAE），与主 Simulink 求解器共享同一步长时，开关事件会迫使求解器无限压缩步长以满足 DAE 精度，最终触发最小步长警告。

两者叠加，系统刚性比（最快模态 / 最慢模态）远超 ode45 等非刚性求解器的稳定域，导致步长崩溃。

### 12.3 三级修复方案

按优先级排序，P1 和 P2 是必做项。

#### P1（必做）：为 Simscape 启用局部求解器

这是消除 DAE 刚性的根本手段。操作路径如下：

1. 双击打开"电机模型"子系统。
2. 找到内部的 **Solver Configuration** 块，双击打开参数对话框。
3. 按下表设置：

| 参数 | 原值（默认） | 修改为 |
|------|-------------|--------|
| Use local solver | ☐ 未勾选 | ☑ 勾选 |
| Solver | — | `Backward Euler`（最鲁棒）|
| Sample time | — | `5e-6`（即 5 μs = T_s/10）|

启用局部求解器后，Simscape 物理网络将以固定步长 5 μs 独立积分，不再倒逼主求解器压步。5 μs 为 T_s/10，每个 PWM 周期内有 10 个积分点，精度与速度的最佳折中。

#### P2（必做）：将主求解器改为刚性求解器

操作路径：**Simulation → Model Configuration Parameters → Solver 标签**

| 参数 | 修改为 |
|------|--------|
| Solver（求解器类型） | `ode23t`（推荐；极高刚性场景改用 `ode15s`）|
| Max step size（最大步长） | `1.25e-5`（= T_s/4 = 12.5 μs）|
| Relative tolerance（相对误差容限） | `1e-3`（保持原值）|
| Absolute tolerance（绝对误差容限） | `1e-5` |
| Min step size（最小步长） | `auto`（**不要手动填写极小值**）|

说明：

1. `ode23t` 是低阶梯形隐式法，对中等刚性表现良好，且不引入虚假阻尼。
2. `ode15s` 是变阶 BDF 法，适用于更高刚性，但偶尔对 Simscape 系统有收敛偏差。
3. 最大步长设为 T_s/4 是为了确保每个 PWM 周期内有足够的采样点，不遗漏开关事件。

#### P3（强烈建议）：为逆变器开关添加缓冲器（Snubber）

理想开关在 Simscape 中是瞬时跳变，是步长崩溃的另一来源。为 PWM Generator 下游的 IGBT/MOSFET 开关添加缓冲器可以平滑跳变：

操作路径：双击 IGBT 或 MOSFET 模块 → 找到 **Snubber** 相关参数：

| 参数 | 建议值 |
|------|--------|
| Snubber resistance Rs | `1e4 Ω`（10 kΩ）|
| Snubber capacitance Cs | `1e-9 F`（1 nF）|

注意：若缓冲器参数过小（Rs 太小或 Cs 太大），反而会在高频引入新的刚性模态，建议先用上表数值。

### 12.4 修复后的验收标准

完成 P1+P2 后，重新运行仿真，应满足：

1. 仿真不再报"最小步长违规"警告。
2. 仿真速度明显快于崩溃前（步长不再无效压缩）。
3. 在 Scope 中可观察到正常的 omega_m 上升曲线，没有发散或数值噪声。

若 P1+P2 完成后问题仍存在，按如下顺序排查：

1. 将 `ode23t` 改为 `ode15s`，最大步长缩小到 T_s/8 = 6.25e-6。
2. 确认 Solver Configuration 块的局部步长已生效（勾选 Use local solver）。
3. 完成 P3（添加 Snubber），减少理想开关跳变对 DAE 的冲击。
4. 检查模型中是否存在代数环（Algebraic Loop）——若 MATLAB 报告代数环，需要在环路上插入 Memory 块或 Unit Delay 打断。
