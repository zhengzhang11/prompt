## 仓库结构

```
robot-servo-sim/
├── 00_frozen/          ← 需求冻结，只读，S1 维护（题目原文 + 唯一参数真相源）
├── 01_prompts/         ← Prompt 原文（每人一个子目录）
├── 02_responses/       ← 模型原始输出（一字不改）
├── 03_checks/          ← 人工校验笔记（含可信度标签）
├── 04_simulink/        ← Simulink 模型文件（.slx + init_params.m）
```

## 快速参考：每日工作节奏

```bash
# 开工前（必做）
git pull

# 工作中（每次 PCR 完成后立即执行）
git add .
git commit -m "[Sx] NNN 任务名 vX — 一句话成果"
git push origin main          # 过程记录直接推 main

# 关键结论（影响他人工作的输出）→ 开 PR
git checkout -b Sx/任务名
git push origin Sx/任务名
# 去 GitHub 网页 → Compare & pull request → 指定 reviewer → 等 1 人 Approve 后 merge
```
