# 构建方案1 Prompt

## 背景说明

考虑到这两年 AI 辅助电气及电机领域仿真设计的功能日益强大，现在电气传动系统老师要我们根据大模型做一个仿真，要求 5 个人一起完成一个项目设计大作业，利用 AI 生成具体的高可靠性的方案、给出详细的可复现的完整仿真技术操作步骤、以及最终的实验数据记录及报告撰写；并且老师让我们做 PPT 来介绍我们用 AI 实现仿真设计的工作流程。大作业要求如上文件，经过讨论，我们选择差速驱动轮电机系统（A 题）作为小组选题，小组成员的分工如下：

| 角色 | 职责 | 主要任务 |
|------|------|---------|
| S1 总架构师 / 提示词工程 PM | 把控顶层设计，写顶层提示词，做技术决策仲裁，管理 Prompt 仓库 | ①②⑦ 主导，全程审核 |
| S2 力学与电机层工程师 | L1+L2 层：负载折算、电机选型、传动设计、BOM 成本核算 | ③（力学/电机部分）、④（Simscape 机械子模型） |
| S3 主电路与电源层工程师 | L3 层：主电路拓扑、器件选型、制动方案、保护设计 | ③（主电路部分）、④（功率电子子模型） |
| S4 控制算法与建模工程师 | L4+L5 层：PI 整定、Simulink 控制框图、参数化脚本 | ③（控制器部分）、④（控制系统子模型）主力 |
| S5 集成测试与文档工程师 | L6 层：集成、工况跑通、数据记录、报告与 PPT 主笔 | ⑤⑥⑦ 主导 |

其中对于文档中的表格参数要求，我们令 ABCD=0000（消除学号影响）。

## 任务指令

我现在担任 S1——总架构师职责，需要你作为电机运动控制系统的资深架构师，以行业内顶尖、满分的标准，按照我下面的要求撰写一份详细、全面、具有高可靠性与鲁棒性的技术架构方案，以**全面性、可实现性**为最高原则。

### 1. 系统整体架构与分系统拆解

首先在详细、深度解读完题目要求后，设计这个系统的整体架构；将题设中具体的目标要求进行清晰明确的拆解，并根据上面的人员分工及行业内标准化的项目步骤，转化为可执行的分系统方案。

### 2. 力学与电机层设计

在力学与电机层设计时，综合题目指标及 MathWorks 等开源社区的高质量流行资源贡献（例如 [BLDC Position Control](https://ww2.mathworks.cn/help/sps/ug/bldc-position-control.html?utm_source=chatgpt.com) 等），选择市面上流行的、Simulink 开源资源丰富的电机类型、传感器类型及辅助器件类型；并进行负载端及传动系统的物理量计算与数学模型搭建，要求在各物理量适用于选定的开源电机 Simulink 模型下，数学建模过程有严谨全面的数学推导，详情见文档中的第三大章表格中的 1、3、4、5 任务模块。

### 3. 主电路（电力电子）与电源层设计

主电路设计要求以**成熟性**为第一原则：

- 优先选取模块兼容常见 MATLAB/Simulink 版本，优先按 R2023a 考虑
- 不要使用版本敏感的库或特殊工具箱
- 优先在网络上寻找高质量、低复杂性的通用模版——例如最流行最稳定的电源设计拓扑，最常见最稳定的电力电子电机驱动电路拓扑

### 4. 控制器设计

控制器部分应给出电机的**单闭环和双闭环 PI 对比设计**，在综合第 1、2、3 点的具体工况后，给出清晰、明确的针对此工况的单/双闭环链路，要求能将第 1、2、3 点提出的方案构想清晰地串在一起，形成高可靠性、高复现性的闭环系统。同时给出整个系统需要整定的控制参数及**详细的参数整定步骤**。

### 5. 设计风格约束

整个方案设计风格为：

- 尽可能多用**成熟的、流行的、开源案例多的**技术方案，保证仿真项目能较小代价跑通、落地
- **切忌**强调创新性而增加仿真系统的可实现性难度
- 你的这个总体方案设计非常重要，将直接决定团队的技术走向，请务必严肃、认真对待，生成**高质量、高准确、零幻觉**的方案设计
